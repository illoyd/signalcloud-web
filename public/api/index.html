Not only can you ask and answer from just about anywhere, but you can connect with SignalCloud from just about anywhere too.

Basics

The SignalCloud REST API allows you to query meta-data about your account, phone numbers, phone books, templates, and tickets. You can also open new tickets as well as manage existing tickets.

Since the API is based on REST principles, it’s very easy to write and test applications. You can use your browser to access URLs, and you can use pretty much any HTTP client in any programming language to interact with the API.

REST API end-point

SignalCloud’s main REST API end-point can be found at the following location: 

https://app.signalcloudapp.com

If you have a dedicated instance, replace app with your instance name. For instance, if your instance was sushi, your REST API end-point would be the following: 

https://sushi.signalcloudapp.com

For simplicity, all our examples will assume you are using the standard REST API end-point, but everything described here applies to both standard accounts as well as dedicated instances.

Security

SignalCloud takes great pains to protect your data.

Authenticating

To authenticate with the SignalCloud REST API, you must specify either HTTP Basic or Digest Authentication, using your account’s username and password. Don’t worry - since we use SSL, your username and password are protected. 

https://username:password@app.signalcloudapp.com

Managing users

You can manage your user accounts using the SignalCloud website.
